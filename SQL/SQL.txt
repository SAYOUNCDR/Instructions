# Tf is Auto-Commit?
Auto-Commit is a feature in SQL that automatically saves changes to the database after each individual statement is executed. 
This means that once a statement is executed, the changes are immediately permanent and cannot be rolled back.

# Why is Auto-Commit important? 
Auto-Commit is important because it ensures that changes to the database are saved immediately, which can help prevent data loss in case of unexpected errors or crashes.
However, it can also lead to unintended consequences if not used carefully, as it may commit changes that were not intended to be permanent.


# How to disable Auto-Commit?
To disable Auto-Commit, you can use the following command:
SET autocommit = OFF;

# How to enable Auto-Commit?
To enable Auto-Commit, you can use the following command:
SET autocommit = ON;


# What is a Transaction?
A transaction is a sequence of one or more SQL statements that are executed as a single unit of work.

A check constraint is a database constraint that ensures that the values in a column meet a specific condition.

# Joins
1. Inner Join - returns only the rows that have matching values in both tables. If there is no match, the row is not included in the result set. i.e only the rows that have a match in both tables will be included in the result set, and if there is no match, the row will be excluded from the result set.
2. Left Join - returns all the rows from the left table and the matching rows from the right table. If there is no match, the result is NULL on the right side. i.e all rows from the left table will be included in the result set, and if there is no match in the right table, the columns from the right table will contain NULL values.
3. Right Join - returns all the rows from the right table and the matching rows from the left table. If there is no match, the result is NULL on the left side. i.e all rows from the right table will be included in the result set, and if there is no match in the left table, the columns from the left table will contain NULL values.
4. Full Outer Join - returns all the rows when there is a match in either left or right table. If there is no match, the result is NULL on the side that does not have a match. i.e all rows from both tables will be included in the result set, and if there is no match in either table, the columns from that table will contain NULL values.


Create two tables- customers and orders

CREATE TABLE customers (
	id INT PRIMARY KEY,
	first_name VARCHAR(50) NOT NULL,
	last_name VARCHAR(50)
)

CREATE TABLE orders (
	id INT PRIMARY KEY,
	order_date DATE,
	order_amount DECIMAL(10,2),
	customer_id INT,
	FOREIGN KEY (customer_id) REFERENCES customers(id)
)

# Inner Join example
SELECT customers.first_name, customers.last_name, orders.order_date, orders.order_amount
FROM customers INNER JOIN orders ON customers.id = orders.customer_id;

# Left Join example
SELECT customers.first_name, customers.last_name, orders.order_date, orders.order_amount
FROM customers LEFT JOIN orders ON customers.id = orders.customer_id;

# Right Join example
SELECT customers.first_name, customers.last_name, orders.order_date, orders.order_amount
FROM customers RIGHT JOIN orders ON customers.id = orders.customer_id;

# Full Outer Join example - this will not work in MySQL as it does not support full outer join, but in other databases like PostgreSQL or SQL Server it will work
SELECT customers.first_name, customers.last_name, orders.order_date, orders.order_amount
FROM customers FULL OUTER JOIN orders ON customers.id = orders.customer_id;

# Aggregate functions in SQL
1. COUNT() - returns the number of rows that match a specified condition.
SELECT COUNT(*) FROM customers WHERE first_name = 'John' AS john_count - this return number of rows that match the condition
SELECT COUNT(order_amount) FROM orders WHERE order_amount IS NOT NULL AS non_null_order_amounts - this return number of non-null values in the specified column that match the condition


2. SUM() - returns the total sum of a numeric column.
SELECT SUM(order_amount) FROM orders AS total_sales - this return the total sum of the order_amount column
SELECT SUM(order_amount) FROM orders WHERE order_date >= '2025-01-01' AND order_date <= '2025-01-31' AS january_sales - this return the total sum of the order_amount column for orders placed in January 2025

3. AVG() - returns the average value of a numeric column.
SELECT AVG(order_amount) FROM orders AS average_order_amount - this return the average value of the order_amount column
SELECT AVG(order_amount) FROM orders WHERE order_date >= '2025-01-01' AND order_date <= '2025-01-31' AS average_january_order_amount - this return the average value of the order_amount column for orders placed in January 2025

4. MIN() - returns the minimum value of a column.
SELECT MIN(order_amount) FROM orders AS minimum_order_amount - this return the minimum value of the order_amount column
SELECT MIN(order_amount) FROM orders WHERE order_date >= '2025-01-01' AND order_date <= '2025-01-31' AS minimum_january_order_amount - this return the minimum value of the order_amount column for orders placed in January 2025

5. MAX() - returns the maximum value of a column.
SELECT MAX(order_amount) FROM orders AS maximum_order_amount - this return the maximum value of the order_amount column
SELECT MAX(order_amount) FROM orders WHERE order_date >= '2025-01-01' AND order_date <= '2025-01-31' AS maximum_january_order_amount - this return the maximum value of the order_amount column for orders placed in January 2025

-- Logical Operators in SQL
1. AND - returns true if both conditions are true.
SELECT * FROM customers WHERE first_name = 'John' AND last_name = 'Doe';

2. OR - returns true if at least one of the conditions is true.
SELECT * FROM customers WHERE first_name = 'John' OR last_name = 'Doe';

3. NOT - returns true if the condition is false.
SELECT * FROM customers WHERE NOT first_name = 'John';

4. IN - returns true if a value matches any value in a list.
SELECT * FROM customers WHERE first_name IN ('John', 'Jane', 'Doe');


-- Wild card characters in SQL
1. % - represents zero or more characters.
SELECT * FROM customers WHERE first_name LIKE 'J%'; - this will return all customers whose first name starts with 'J'

SELECT * FROM customers WHERE first_name LIKE '%n'; - this will return all customers whose first name ends with 'n'

SELECT * FROM customers WHERE first_name LIKE '%oh%'; - this will return all customers whose first name contains 'oh' anywhere in the name

2. _ - represents a single character.
SELECT * FROM customers WHERE first_name LIKE 'J_n'; - this will return all customers whose first name starts with 'J', followed by any single character, and ends with 'n' (e.g.,  'Jan', 'Jen')

SELECT * FROM customers WHERE first_name LIKE '_ohn'; - this will return all customers whose first name has any single character followed by 'ohn' (e.g., 'John', 'Kohn', 'Lohn')

SELECT * FROM customers WHERE first_name LIKE 'J__n'; - this will return all customers whose first name starts with 'J', followed by any two characters, and ends with 'n' (e.g., 'John', 'Jann', 'Jenn')


NOTE - We can use both % and _ wildcards together in a LIKE pattern to create more complex search criteria. For example:

SELECT * FROM employee where first_name like "_aa%"; - this will return all employees whose first name has any single character followed by 'aa' and then any sequence of characters (e.g., 'Saara', 'Raahul', 'Kaashif')




// Ordder BY clause in SQL
The ORDER BY clause is used to sort the result set of a query in either ascending (ASC) or descending (DESC) order based on one or more columns.
example-
SELECT * FROM customers ORDER BY first_name ASC; - this will return all customers sorted by their first name in ascending order (A to Z)
SELECT * FROM customers ORDER BY last_name DESC; - this will return all customers sorted by their last name in descending order (Z to A)


// use order by with multiple columns
SELECT * FROM orders ORDER BY amount DESC, order_date ASC; - this will return all orders sorted by amount in descending order, and if there are multiple orders with the same amount, they will be sorted by order date in ascending order.

// Limit clause in SQL
The LIMIT clause is used to specify the maximum number of rows to return in a result set.
example-
SELECT * FROM customers LIMIT 5; - this will return the first 5 rows

SELECT * FROM customers ORDER BY id ASC LIMIT 5; - this will return the first 5 rows sorted by id in ascending order

//  what is offset in Limit clause?
The OFFSET clause is used in conjunction with the LIMIT clause to specify the number of rows to skip before starting to return rows from the query result. 
It is often used for pagination purposes, allowing you to retrieve a specific subset of rows from a larger result set.
example-
SELECT * FROM customers ORDER BY id ASC LIMIT 10, 10 ; - this will skip the first 20 rows and return the next 10 rows

like first page shows rows 1-10, second page shows rows 11-20, third page shows rows 21-30 and so on


// Union and Union All in SQL
1. UNION - combines the result sets of two or more SELECT statements and returns only distinct rows from the combined result set.

// note - both tables must have the same number of columns and compatible data types for the corresponding columns in the SELECT statements.
example- 
SELECT first_name FROM customers
UNION
SELECT product_name FROM products; - this will return a combined list of first names from the customers table and product names from the products table, but only distinct values will be included in the result set.


2. UNION ALL - combines the result sets of two or more SELECT statements and returns all rows from the combined result set, including duplicates.
example-
SELECT first_name FROM customers 
UNION ALL
SELECT product_name FROM products; - this will return a combined list of first names from the customers table and product names from the products table, including all duplicates in the result set.


// self join in SQL
A self join is a type of join that allows you to join a table to itself. 
It is used when you want to compare rows within the same table or when you want to retrieve related data from the same table.

example-
CREATE TABLE employees (
	id INT PRIMARY KEY,
	first_name VARCHAR(50),
	last_name VARCHAR(50),
	manager_id INT,
	FOREIGN KEY (manager_id) REFERENCES employees(id)
)

SELECT e1.first_name AS employee_name, e2.first_name AS manager_name
FROM employees e1
LEFT JOIN employees e2 ON e1.manager_id = e2.id; - this will return a list of employees along with their managers' names. If an employee does not have a manager (i.e., manager_id is NULL), the manager_name will be returned as NULL in the result set.
